# START-HERE

If you are new to the public UMG Block Library, use this order:

1. `README.md`
2. `META/PUBLIC-SCOPE.md`
3. `META/PRIVATE-EXCLUSIONS.md`
4. `META/CONTENT-STATUS.md`
5. `AI/DOCTRINE/`
6. `AI/SCHEMAS/`
7. `HUMAN/GLOSSARY/`

This first population pass includes known-safe doctrine, schema templates, a curated direct sleeve catalog, and baseline public-safe MOLT block libraries.
Other designed folders are intentionally left as placeholders until they are ready to populate honestly.

## Fast orientation
If you are trying to understand how the library fits together, use this path:

1. `AI/MANIFESTS/README.md` for direct/source-oriented machine catalogs
2. `sleeves/manifests/README.md` for the curated public/package-facing sleeve catalog
3. `HUMAN/SLEEVES/categories/README.md` for the human browsing shelf
4. `HUMAN/NEOSTACKS/README.md` for workflow lanes
5. `HUMAN/NEOBLOCKS/README.md` for bundled middle-layer units
6. `HUMAN/MOLT-BLOCKS/README.md` for the smallest idea/rule families

Remember the stack:

```text
MOLT Block
→ NeoBlock
→ NeoStack
→ Sleeve
```

And remember the lane split:
- `AI/` = machine-readable shelf
- `HUMAN/` = human-readable shelf
- `sleeves/` = public/package-facing machine artifact lane
