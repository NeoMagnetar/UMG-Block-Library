# NEED.035 - Ambiguity resolved

**Type:** NEED
**Category:** REQUIREMENT_CLARITY
**Group Description:** What must be true about requirements and scope before design or build begins.
**Status:** active

## Definition
Unclear or contradictory requirements must be clarified before work proceeds.

## When to use
What must be true about requirements and scope before design or build begins.

## Examples
- open ODRs that block design must be resolved first
- conflicting block definitions must be reconciled before compilation
- ambiguous requirements must be clarified before implementation
