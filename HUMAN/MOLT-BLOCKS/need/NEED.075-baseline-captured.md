# NEED.075 - Baseline captured

**Type:** NEED
**Category:** OBSERVABILITY_BASELINE
**Group Description:** What must be true about logging, metrics, and monitoring before a system goes live.
**Status:** active

## Definition
Normal system behavior must be characterized before deviations can be detected as anomalies.

## When to use
What must be true about logging, metrics, and monitoring before a system goes live.

## Examples
- baseline metrics must be recorded before anomaly detection is trusted
- expected compilation time must be profiled before regression alert
- normal error rate must be known before alerting threshold is set
