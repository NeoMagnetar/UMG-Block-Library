# NEED.027 - Role assigned

**Type:** NEED
**Category:** ACCESS_AUTHORIZATION
**Group Description:** What must be true about identity and permission before action is permitted.
**Status:** active

## Definition
The actor must have a defined role that specifies what they are permitted to do.

## When to use
What must be true about identity and permission before action is permitted.

## Examples
- admin role must be assigned before config change
- Envoy role must be declared before sleeve activation
- reviewer role must be granted before approval action
