# NEED.069 - Error handling agreed

**Type:** NEED
**Category:** INTERFACE_CONTRACT
**Group Description:** What must be true about interfaces and contracts before systems can connect.
**Status:** active

## Definition
How failures will be signaled and handled must be documented before the interface is used.

## When to use
What must be true about interfaces and contracts before systems can connect.

## Examples
- error codes must be defined before client handles them
- governance fail-fast behavior must be specified before compilation
- retry semantics must be agreed before queue consumer is built
