# NEED.084 - Provenance established

**Type:** NEED
**Category:** DATA_QUALITY_BASELINE
**Group Description:** What must be true about data quality before analysis or decisions can rely on it.
**Status:** active

## Definition
The origin and chain of custody of data must be known before it is relied upon for decisions.

## When to use
What must be true about data quality before analysis or decisions can rely on it.

## Examples
- block source must be traceable before it is used in governance
- data lineage must be documented before regulatory report
- Decision Trace must record block origin before audit
