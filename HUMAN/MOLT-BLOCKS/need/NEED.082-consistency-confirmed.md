# NEED.082 - Consistency confirmed

**Type:** NEED
**Category:** DATA_QUALITY_BASELINE
**Group Description:** What must be true about data quality before analysis or decisions can rely on it.
**Status:** active

## Definition
Data must not contain internal contradictions within or across sources before it is trusted.

## When to use
What must be true about data quality before analysis or decisions can rely on it.

## Examples
- block definitions must not conflict before library is used
- config values must not contradict each other before deploy
- governance rules must not conflict before enforcement
