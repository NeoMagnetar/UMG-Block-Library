# NEED.087 - Rollback plan ready

**Type:** NEED
**Category:** DELIVERY_READINESS
**Group Description:** What must be true about delivery pipelines and processes before releasing.
**Status:** active

## Definition
A way to undo the delivery must exist and be tested before proceeding.

## When to use
What must be true about delivery pipelines and processes before releasing.

## Examples
- previous version must be tagged before new deploy
- governance rollback procedure must be documented before rule change
- database migration must be reversible before apply
