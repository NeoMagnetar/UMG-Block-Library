# NEED.040 - Impact quantified

**Type:** NEED
**Category:** PROBLEM_DEFINITION
**Group Description:** What must be true about a problem before diagnosis or resolution begins.
**Status:** active

## Definition
The severity and scope of the problem must be measurable before priority is set.

## When to use
What must be true about a problem before diagnosis or resolution begins.

## Examples
- number of affected users must be known before escalation
- data loss extent must be measured before recovery plan
- block failure rate must be measured before redesign
