# NEED.080 - Policy version controlled

**Type:** NEED
**Category:** POLICY_DEFINITION
**Group Description:** What must be true about policies and rules before they can be enforced.
**Status:** active

## Definition
Changes to policy must be tracked with clear versioning before multiple versions can coexist or evolve.

## When to use
What must be true about policies and rules before they can be enforced.

## Examples
- governance stack must be tagged before canon update
- policy document must be committed to source control
- MOLT invariants must be versioned before v1 is declared
