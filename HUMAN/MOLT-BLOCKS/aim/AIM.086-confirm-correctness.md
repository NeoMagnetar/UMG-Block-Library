**Type:** AIM
**Category:** VERIFICATION_OUTCOMES

## Definition
To prove that a system or output does exactly what it claims to do.

## When to use
Aims oriented toward proving, confirming, and certifying.

## Examples
- verifying a RuntimeSpec matches the compiled CIR
- confirming a function returns expected output
- proving a policy has no gaps
