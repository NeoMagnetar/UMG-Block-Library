# HUMAN SLEEVES CATEGORIES

## What this section is
This section is the browsing shelf for human-readable sleeve categories. Categories organize sleeves into approachable groups so readers can start with a role, domain, or operating style before drilling into the deeper layers.

## What this section is not
This is not the authoritative machine runtime catalog, and it is not a guarantee that every human-readable sleeve summary has a public runnable machine JSON sleeve.

## Who should use this
- Humans
- maintainers
- future contributors
- AI agents that need the public-readable browsing map

## Canonical source rule
Current status: human-readable explanation and browsing shelf.

## How this maps to nearby sections
- `HUMAN/SLEEVES/` contains readable sleeve summaries for people.
- `sleeves/manifests/` contains the public/package-facing machine catalog.
- `AI/MANIFESTS/` contains the direct/source-oriented machine catalog.
- `HUMAN/NEOSTACKS/` and `HUMAN/NEOBLOCKS/` are the intended middle layers beneath sleeves.
- `HUMAN/MOLT-BLOCKS/` contains the smallest conceptual families.

## Current status

## Sleeve lane clarification
Two sleeve shelves exist on purpose:

- `HUMAN/SLEEVES/` is the readable sleeve summary shelf for people.
- top-level `sleeves/` is the public/package-facing machine artifact lane.

Human sleeve pages may be richer or broader than the public machine catalog.
Not every human sleeve summary is automatically a runnable public JSON sleeve.
Use the machine catalogs to confirm what is promoted.

## Navigation goal
Every sleeve category should eventually support this browsing path:

```text
category → sleeve → NeoStack → NeoBlock → MOLT block
```

Some categories are already useful browsing shelves.
Some remain placeholders while the public-safe bridge layers are normalized.
